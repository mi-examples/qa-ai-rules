# Test Case Rules

Follow these rules strictly when generating test cases.

## Content Rules

- No duplicate checks
- Only include role-specific steps when functionality differs by role
- Ensure the final document is concise and free of duplicate checks

## Structure Rules

- Group test cases by logical blocks
- Number test cases sequentially (TC-01, TC-02, …)
- Start the file with an `Overview` block
- Add a `Notes` section immediately after `Overview`

## Each Test Case Must Contain

- Title
- Priority (`High` / `Medium` / `Low`)
- Steps (numbered)
- Expected Result

## Formatting Rules

- Written in Markdown
- Use headings for sections
- Test case title format: `### TC-01 – Title` (no bold markers around the ID)
- Use numbered lists for steps
- Keep descriptions compact but clear
- Easy to read for someone seeing the document for the first time
